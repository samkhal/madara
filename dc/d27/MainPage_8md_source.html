6 You are accessing the core C++ API documentation for one of the most feature-rich
7 and fastest distributed application development libraries in the world. MADARA
8 has been primarily used for building distributed artificial intelligence and
9 knowledge bases in robotics that are resilient to extreme outdoor environments,
10 network disconnections, and problematic sensors and actuators.
11 
12 MADARA facilitates development and even formal verification by providing a unique,
13 intuitive interface that integrates a distributed knowledge base into C++ programs
14 through the inclusion of a knowledge base, a networking transport, and C++ variables
15 called containers that look and feel like normal object-oriented C++ code, but that
16 send and receive knowledge modifications over the network. MADARA includes advanced threading
17 and networking features like bandwidth enforcement, network filtering, and thread
18 hertz enforcement. It also includes hundreds of other quality-of-service and optimization
19 features that allow MADARA to operate fast and efficiently, even on ARM and other reduced
20 instruction set CPUs.
21 
22 For new users, you may want to try the C++ guide series, which walks developers through the MADARA API
23 and basic usage.

39 Places in the API documentation to start looking for information on:
40 
41 1. Using The Knowledge Base
42  1. Browse object-oriented containers via madara::knowledge::containers classes (preferred)
43  2. Use direct Knowledge Base functions via madara::knowledge::KnowledgeBase
44 2. Transfering Knowledge / Networking
45  1. Browse the builtin transports for UDP, Broadcast or Multicast in madara::transport
46  2. Make your own transport by extending madara::transport::Base
47 3. Logging information to files or console
48  1. Browse verbose logging options for MADARA internals via madara::logger::LogLevels set on madara::logger::Logger via madara::logger::Logger::set_level
49  2. Use the global logger (madara::logger::global_logger) by including madara/logger/GlobalLogger.h
50  3. Create your own logger and enumerations by creating a new instance of madara::logger::Logger. You don't have to extend it.
51  4. Use high performance logging macros like madara::logger::madara_logger_ptr_log with the global_logger or madara::logger::madara_logger_log with your own logger instance. These only perform function calls if the log level is appropriate (super fast)
52 4. Calling functions from inside the KaRL interpreter
53  1. Define custom C++ functions and bind them to function names in KaRL via madara::knowledge::KnowledgeBase::define_function
54  2. Evaluate such functions in KaRL by calling madara::knowledge::KnowledgeBase::evaluate
55 5. Using Event-Based Programming
56  1. Create custom on-send events by extending madara::filters::AggregateFilter and adding them to transports with madara::transport::QoSTransportSettings::add_send_filter(filters::AggregateFilter * filter)
57  2. Create custom on-receive events by extending madara::filters::AggregateFilter and adding them to transports with madara::transport::QoSTransportSettings::add_receive_filter(filters::AggregateFilter * filter)
58  2. Create custom on-rebroadcast events by extending madara::filters::AggregateFilter and adding them to transports with madara::transport::QoSTransportSettings::add_rebroadcast_filter(filters::AggregateFilter * filter)
59 6. Enforcing Bandwidth Limits
60  1. Set bandwidth limits in QoSTransportSettings via madara::transport::QoSTransportSettings::set_send_bandwidth_limit or base limits on total throughput over the transport with madara::transport::QoSTransportSettings::set_total_bandwidth_limit
61  2. Create custom filters for madara::transport::QoSTransportSettings::add_send_filter(filters::AggregateFilter * filter) by extending madara::filters::AggregateFilter
62 7. Creating more realistic tests
63  1. Set drop rate and drop type via madara::transport::QoSTransportSettings::update_drop_rate
64 8. Working with networks
65  1. Add built-in transports via madara::knowledge::KnowledgeBase::attach_transport(const std::string &, transport::TransportSettings & settings)
66  2. Add custom transports by extending madara::transport::Base and attaching the transport via madara::knowledge::KnowledgeBase::attach_transport(madara::transport::Base *)
